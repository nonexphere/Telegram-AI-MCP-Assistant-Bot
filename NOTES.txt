# Telegram AI Assistant Bot - Comprehensive Project Report

## Project Overview
The Telegram AI Assistant Bot, named Cortex-R, is a reasoning-driven AI agent that interacts with users via Telegram. It leverages multiple Model Context Protocol (MCP) servers to perform various tasks such as web searches, document processing, mathematical calculations, and Google services integration (e.g., Gmail and Google Sheets).

## Core Components

### 1. Telegram Bot Integration
- **File**: `telegram_agent.py`
- **Role**: Handles user messages and commands via Telegram.
- **Features**:
  - Uses the `python-telegram-bot` library.
  - Processes user queries using the Cortex-R agent.
  - Manages commands like `/start` and `/help`.

### 2. Agent Loop
- **File**: `core/loop.py`
- **Role**: Manages the main execution loop of the agent.
- **Features**:
  - Handles perception, planning, and tool execution.
  - Manages session state and tool call tracking.
  - Implements the agent's strategy for decision-making.

### 3. Session Management
- **File**: `core/session.py`
- **Role**: Manages communication with multiple MCP servers.
- **Features**:
  - Uses `stdio` and `sse` transport protocols.
  - Dynamically discovers tools from MCP servers.
  - Handles tool calls and session management.

### 4. Strategy
- **File**: `core/strategy.py`
- **Role**: Implements planning strategies for decision-making.
- **Features**:
  - Supports different strategies like conservative, retry-once, and explore-all.
  - Wraps around the decision-making logic.
  - Adds planning context such as past failures, retries, and agent profile.

## Modules

### 1. Action
- **File**: `modules/action.py`
- **Role**: Handles parsing function call strings into tool names and argument dictionaries.
- **Features**:
  - Defines the `ToolCallResult` class.
  - Parses function call strings into structured data.

### 2. Decision
- **File**: `modules/decision.py`
- **Role**: Generates the next step plan for the agent.
- **Features**:
  - Uses the `ModelManager` to generate text based on perception, memory items, and tool descriptions.
  - Returns a structured plan for either tool usage or a final answer.

### 3. Memory
- **File**: `modules/memory.py`
- **Role**: Manages embedding-based semantic memory using FAISS.
- **Features**:
  - Stores and retrieves `MemoryItem` objects.
  - Uses local embedding server for vectorization.
  - Filters memory based on type, tags, and session.

### 4. Model Manager
- **File**: `modules/model_manager.py`
- **Role**: Handles model management for text generation.
- **Features**:
  - Supports different model types such as Gemini and Ollama.
  - Includes methods for generating text based on prompts.

### 5. Perception
- **File**: `modules/perception.py`
- **Role**: Uses LLMs to extract structured information from user input.
- **Features**:
  - Extracts intent, entities, and tool hints.
  - Uses the `ModelManager` to generate text based on user input and available tools.
  - Parses the response into a `PerceptionResult` object.

### 6. Tools
- **File**: `modules/tools.py`
- **Role**: Provides utility functions for managing and describing tools.
- **Features**:
  - Summarizes tools for LLM prompts.
  - Filters tools based on hints.
  - Creates a map of tool names to tool objects for quick lookup.

## Workflow

### 1. Initialization
- **Description**: The bot loads MCP server configurations from `profiles.yaml`.
- **Implementation**: `telegram_agent.py`
- **Features**:
  - Initializes the `MultiMCP` dispatcher to discover tools from all configured MCP servers.

### 2. User Interaction
- **Description**: Users interact with the bot via Telegram.
- **Implementation**: `telegram_agent.py`
- **Features**:
  - The bot processes user messages and determines the required tools to fulfill the task.

### 3. Task Execution
- **Description**: The agent uses perception to understand the user query.
- **Implementation**: `core/loop.py`
- **Features**:
  - Plans the next action using the defined strategy.
  - Executes the required tool from the appropriate MCP server.
  - Retrieves results and responds to the user.

### 4. Memory Management
- **Description**: Stores tool outputs and user queries as memory items.
- **Implementation**: `modules/memory.py`
- **Features**:
  - Retrieves relevant memories to assist in task-solving.
